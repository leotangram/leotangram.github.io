Soluciones para nuestra tarea Equi

  Programación
Por
Tomasz Waleń
25 de marzo de 2011

Para los programadores que practican sus habilidades de codificación, o que se preparan para tomar una evaluación de Codility, ofrecemos una serie de lecciones gratuitas, desafíos y problemas de demostración en el hogar de nuestros programadores . 

¡Una de las tareas más populares es Equi y hasta la fecha hemos evaluado más de 50,000 soluciones a este problema!

El lenguaje más popular utilizado para resolver este problema es Java, aunque C #, PHP y C ++ también son muy populares: 

La descripción del problema es muy corta:

 

El índice de equilibrio de una secuencia es un índice tal que la suma de elementos en índices más bajos es igual a la suma de elementos en índices más altos. Por ejemplo, en una secuencia A:

A [0] = - 7 A [1] = 1 A [2] = 5 A [3] = 2 A [4] = - 4 A [5] = 3 A [6] = 0

3 es un índice de equilibrio, porque:

A [0] + A [1] + A [2] = A [4] + A [5] + A [6]

6 es también un índice de equilibrio, porque:

A [0] + A [1] + A [2] + A [3] + A [4] + A [5] = 0

(La suma de cero elementos es cero) 7 no es un índice de equilibrio, porque no es un índice válido de la secuencia A. Si todavía tiene dudas, aquí hay una definición precisa: El número entero k es un índice de equilibrio de una secuencia A [0], A [1] .., A [n-1] si y solo si 0 <= k y suma (A [0 .. (k-1)]) = suma (A [(k + 1) .. (n-1)]) . Suponga que la suma de cero elementos es igual a cero.

Escribe una función

int equi (int A [], int n)

que, dada una secuencia, devuelve su índice de equilibrio (cualquiera) o -1 si no existe un índice de equilibrio. Supongamos que la secuencia puede ser muy larga.

 

El problema se puede resolver usando varios enfoques, el más común es simplemente seguir la definición de equilibrio:

int equi ( int A [], int n) {
     int k, m, lsum, rsum; 
    para (k = 0 ; k <n; ++ k) {
        lsum = 0 ; rsum = 0 ;
        para (m = 0 ; m <k; ++ m) lsum + = A [m]; 
        para (m = k + 1 ; m <n; ++ m) rsum + = A [m];  
        if (lsum == rsum) return k;
    } 
    return - 1 ;
}

Desafortunadamente, este enfoque tiene dos desventajas:

  • falla en grandes conjuntos de datos de entrada, ya que la complejidad del tiempo es O (n 2 )
  • falla en valores de entrada grandes (por ejemplo, si la matriz de entrada contiene valores como MIN / MAX_INT) debido a los desbordamientos aritméticos

El análisis de la solución detectará tales problemas en el código enviado:

Podemos solucionar el primer problema con un mejor algoritmo y el segundo problema con un mejor tipo de datos (por ejemplo, usar el tipo largo y largo en lugar de int para los cálculos de suma). La observación clave para un mejor tiempo de ejecución es actualizar las sumas izquierda / derecha en tiempo constante en lugar de volver a calcularlas desde cero.

int equi(int arr[], int n) {
    if (n==0) return -1; 
    long long sum = 0;
    int i; 
    for(i=0;i<n;i++) sum+=(long long) arr[i]; 

    long long sum_left = 0;    
    for(i=0;i<n;i++) {
        long long sum_right = sum - sum_left - (long long) arr[i];
        if (sum_left == sum_right) return i;
        sum_left += (long long) arr[i];
    } 
    return -1; 
}
